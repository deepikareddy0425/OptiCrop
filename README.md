# OptiCrop — Smart Agricultural Production Optimization Engine

A complete, ready-to-run Flask + Machine Learning web application that recommends
the best crop to plant given soil nutrients (N, P, K) and climate readings
(temperature, humidity, pH, rainfall). Built to match every Epic/Story in the
project report: EDA, preprocessing, K-Means clustering, model comparison
(Logistic Regression / Decision Tree / Random Forest / KNN), and a Flask web app.

A trained model is **already included** in `model/`, so the app runs immediately
— retraining is optional.

```
OptiCrop/
├── app.py                  # Flask app (routes + prediction)
├── train_model.py          # EDA, preprocessing, K-Means, model training/eval
├── generate_dataset.py     # Creates a synthetic Crop_recommendation.csv
├── requirements.txt
├── data/
│   └── Crop_recommendation.csv
├── model/
│   ├── model.pkl            # trained classifier (best of 4 models)
│   ├── scaler.pkl            # StandardScaler
│   └── label_encoder.pkl     # crop name <-> class index
├── templates/
│   ├── base.html
│   ├── index.html            # Home
│   ├── about.html            # About
│   └── findyourcrop.html     # Prediction form + result
└── static/
    ├── css/style.css
    └── images/                # EDA plots generated by train_model.py
```

## Step-by-step: Run this in VS Code

### 1. Unzip the project
Unzip `OptiCrop.zip` anywhere on your computer, e.g. `Desktop/OptiCrop`.

### 2. Open it in VS Code
- Open VS Code
- `File > Open Folder…` → select the unzipped `OptiCrop` folder

### 3. Install the Python extension (if you don't have it)
- Go to the Extensions panel (left sidebar, or `Ctrl+Shift+X` / `Cmd+Shift+X`)
- Search for **Python** (by Microsoft) and install it

### 4. Open a terminal in VS Code
- Menu: `Terminal > New Terminal` (or `` Ctrl+` ``)

### 5. Create a virtual environment (recommended)
```bash
python -m venv venv
```
Activate it:
- **Windows:** `venv\Scripts\activate`
- **macOS/Linux:** `source venv/bin/activate`

VS Code may prompt "Select Interpreter" — choose the `venv` one shown, or run
`Python: Select Interpreter` from the Command Palette (`Ctrl+Shift+P`) and pick
`./venv/bin/python` (or `.\venv\Scripts\python.exe` on Windows).

### 6. Install dependencies
```bash
pip install -r requirements.txt
```

### 7. (Optional) Regenerate the dataset and retrain the model
A trained model is already included, so you can skip straight to Step 8.
If you want to retrain (e.g. with your own dataset from Kaggle):

```bash
python generate_dataset.py   # only needed if you don't have your own CSV
python train_model.py
```
- To use the **real Kaggle dataset** instead of the synthetic one, download
  `Crop_recommendation.csv` from Kaggle and place it at `data/Crop_recommendation.csv`,
  then just run `python train_model.py` (skip `generate_dataset.py`).
- This prints model comparison metrics (Accuracy/Precision/Recall/F1) for
  Logistic Regression, Decision Tree, Random Forest, and KNN, saves the best
  one to `model/model.pkl`, and writes EDA/evaluation plots to
  `static/images/`.

### 8. Run the Flask application
```bash
python app.py
```
You'll see output like:
```
 * Running on http://127.0.0.1:5000
```

### 9. Open the app in your browser
Click the link in the terminal, or open **http://127.0.0.1:5000** manually.

- **Home** — project overview
- **About** — how the model works
- **Find Your Crop** — enter N, P, K, temperature, humidity, pH, rainfall and
  click **Predict my crop** to get a recommendation with a confidence score.

### 10. Stop the server
Press `Ctrl+C` in the terminal.

## Troubleshooting

| Problem | Fix |
|---|---|
| `ModuleNotFoundError: No module named 'flask'` | Make sure your virtual environment is activated, then re-run `pip install -r requirements.txt` |
| `FileNotFoundError: model files not found` | Run `python generate_dataset.py` then `python train_model.py` |
| Port 5000 already in use | Edit the last line of `app.py` to `app.run(debug=True, port=5001)` |
| Page has no styling | Make sure you're running `app.py` from the project root (Flask needs to find `static/` and `templates/`) so URLs resolve correctly |

## Notes on the dataset

`generate_dataset.py` produces a **synthetic** dataset that mirrors the schema of
the well-known Kaggle "Crop Recommendation Dataset" (N, P, K, temperature,
humidity, ph, rainfall, label — 22 crops) using realistic per-crop parameter
ranges, so the whole pipeline runs end-to-end out of the box. For production
use or a more accurate model, download the real dataset from Kaggle at:

`https://www.kaggle.com/datasets/chitrakumari25/smart-agricultural-production-optimizing-engine`

and drop it in as `data/Crop_recommendation.csv`, then re-run `python train_model.py`.

## Tech stack

- **Backend:** Flask
- **ML:** scikit-learn (Logistic Regression, Decision Tree, Random Forest, KNN, K-Means)
- **Data:** NumPy, Pandas
- **Visualization:** Matplotlib, Seaborn
- **Frontend:** HTML/CSS (Jinja2 templates), no build step required
